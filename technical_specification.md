# Техническое задание

---

## 1. Общие требования

**1.1. Назначение**  
Разработать полнофункциональный интернет-магазин с возможностью просмотра товаров, добавления в корзину, оформления заказа и администрирования.

**1.2. Аудитория**
- Покупатели (неавторизованные и зарегистрированные)
- Администраторы (управление каталогом, заказами, пользователями)

**1.3. Технологический стек (обязательный)**
- **Язык:** Java 17+
- **Фреймворк:** Spring Boot 3.x (MVC, Security, Data JPA)
- **База данных:** PostgreSQL (основная) + H2 (для тестов)
- **Шаблонизатор:** Thymeleaf
- **Сборка:** Maven
- **Фронтенд:** HTML5, CSS3 (Bootstrap 5), JavaScript (без обязательного SPA)
- **Дополнительно:** Spring Validation, Lombok, MapStruct, Liquibase/Flyway

**1.4. Требования к развёртыванию**
- Должна быть возможность запуска через `docker-compose` (app + postgres)
- Порт приложения: 8080
- Конфигурация через `application.yml` и переменные окружения

---

## 2. Функциональные требования

### 2.1. Модуль пользователей
- Регистрация (email, пароль, имя, адрес доставки)
- Вход/выход (Spring Security, сессии или JWT)
- Роли: `ROLE_USER`, `ROLE_ADMIN`
- Личный кабинет: просмотр/редактирование профиля, история заказов

### 2.2. Каталог товаров
- Список товаров с пагинацией и сортировкой (по цене, названию, дате добавления)
- Фильтрация по категории, цене (диапазон), наличию
- Карточка товара: название, цена, описание, изображение (ссылка или загрузка), количество на складе
- Поиск по названию и описанию (регистронезависимый)

### 2.3. Корзина
- Добавление/удаление/изменение количества товара (без авторизации – через сессию/куки)
- После авторизации – корзина сохраняется в БД
- Отображение общей суммы
- Очистка корзины

### 2.4. Заказы
- Оформление заказа (выбор адреса доставки, способа оплаты)
- Проверка остатков на складе перед оформлением
- После оформления: списание товаров, создание заказа со статусом `CREATED`
- Статусы заказа: `CREATED`, `PAID`, `SHIPPED`, `DELIVERED`, `CANCELLED`
- Пользователь может отменить заказ только в статусе `CREATED`
- Просмотр истории заказов с деталями

### 2.5. Административная панель (доступна только для ROLE_ADMIN)
- CRUD категорий товаров
- CRUD товаров (загрузка изображения на сервер или облако)
- Просмотр всех заказов с фильтром по статусу и пользователю
- Изменение статуса заказа
- Блокировка/разблокировка пользователей

## 3. Нефункциональные требования

- **Безопасность:**
    - пароли хранить в БД в зашифрованном виде (BCrypt)
    - защита от CSRF, XSS, SQL-инъекций
    - после 5 неудачных попыток входа – временная блокировка на 15 минут
- **Производительность:**
    - время ответа API < 300 мс (95% запросов)
    - кэширование списка категорий и популярных товаров (Spring Cache + Caffeine)
- **Логирование:** SLF4J + Logback, отдельный файл для ошибок
- **Тестирование:**
    - юнит-тесты (JUnit 5, Mockito) – покрытие не менее 70% бизнес-логики
    - интеграционные тесты для репозиториев и контроллеров
    - тесты API через MockMvc или TestRestTemplate

## 5. Схема базы данных (основные таблицы)

- `users` (id, email, password, name, address, role, blocked, created_at)
- `categories` (id, name, parent_id – для дерева)
- `products` (id, name, description, price, stock_quantity, image_url, category_id, created_at)
- `carts` (id, user_id – может быть null для сессий)
- `cart_items` (id, cart_id, product_id, quantity)
- `orders` (id, user_id, order_date, total_amount, status, delivery_address, payment_method)
- `order_items` (id, order_id, product_id, quantity, price_at_purchase)

> Индексы: products.name (поиск), orders.user_id, orders.status.

## 6. API (основные эндпоинты)

| Метод | URL                       | Роль       | Описание                        |
|-------|---------------------------|------------|---------------------------------|
| GET   | /products                 | ANY        | список товаров с фильтрацией    |
| GET   | /products/{id}            | ANY        | карточка товара                 |
| POST  | /cart/add/{productId}     | ANY        | добавить в корзину              |
| GET   | /cart                     | ANY        | просмотр корзины                |
| POST  | /cart/update              | ANY        | изменить количество             |
| DELETE| /cart/remove/{itemId}     | ANY        | удалить из корзины              |
| POST  | /orders/create            | USER       | оформить заказ                  |
| GET   | /orders                   | USER       | история заказов                 |
| GET   | /admin/orders             | ADMIN      | все заказы                      |
| PUT   | /admin/orders/{id}/status | ADMIN      | изменить статус заказа          |
| CRUD  | /admin/products           | ADMIN      | управление товарами             |

## 7. Пользовательские сценарии

1. **Гость** заходит на сайт → ищет товар → добавляет в корзину → регистрируется → оформляет заказ → оплачивает (имитация).
2. **Зарегистрированный пользователь** входит → видит сохранённую корзину → оформляет заказ → смотрит историю.
3. **Администратор** входит в админ-панель → добавляет новую категорию → добавляет товар с изображением → меняет статус заказа на "Доставлен".

## 8. Критерии приёмки

- [ ] Приложение успешно собирается командой `mvn clean package`
- [ ] Запускается через `docker-compose up` без ошибок
- [ ] Все CRUD операции работают корректно
- [ ] Корзина сохраняется после перезагрузки браузера (для гостей) и после логина
- [ ] Нельзя оформить заказ с товаром, которого нет в нужном количестве
- [ ] Администратор не может заказать товар как обычный пользователь через подстановку URL
- [ ] Все тесты проходят успешно (`mvn test`)
- [ ] Интерфейс адаптивен (Bootstrap) и понятен на мобильных устройствах
